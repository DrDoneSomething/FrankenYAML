Nested Lists are lists buried in integrations. A good use for this is a settings integration with a list{configure_list_integrations} you wish to break out into individual files/objects.

example: ios/push/categories -- ios is a Setting
Entities location: [left blank - parse script will give it a good home]
Naming scheme for list [REQUIRED IF UNIQUE]:
    Entity Field: value                 Field value to set as name
integration_name:ios/push/categories -> name


ios:
  push:
    categories:
      - name: ArmAlarm
        identifier: 'armalarm'
        actions:
          - identifier: 'ALARM_STAY'
            title: 'Alarm Arm Stay'
          - identifier: 'ALARM_AWAY'
            title: 'Alarm Arm Away'
      - name: CopyNewFiles
        identifier: 'copy_new_files'
        actions:
          - identifier: 'COPY_FILES'
            title: 'Copy Files'
          - identifier: 'IGNORE_FILES'
            title: 'Ignore Message'

Becomes:
[integrations/ios.yaml]
ios:
  push:
    categories: !include_dir_merge_named ../entities/ios_push_categories/

[entities/ios_push_categories/armalarm.yaml]
name: ArmAlarm
identifier: 'armalarm'
actions:
  - identifier: 'ALARM_STAY'
    title: 'Alarm Arm Stay'
  - identifier: 'ALARM_AWAY'
  
[entities/ios_push_categories/CopyNewFiles.yaml]
name: CopyNewFiles
identifier: 'copy_new_files'
actions:
  - identifier: 'COPY_FILES'
    title: 'Copy Files'
  - identifier: 'IGNORE_FILES'
    title: 'Ignore Message'
